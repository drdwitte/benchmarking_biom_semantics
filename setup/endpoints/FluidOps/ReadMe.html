Installation Requirements

Server

Operating System

  • Windows (64-bit only): Windows 7, Windows Server 2012
  • Linux (64-bit only): Ubuntu 14.04, openSUSE 12.1

Java Runtime Environment

  • JRE >=1.8.0_60 64 bit

Other Linux Distributions, different versions of Windows or OS X systems may also work, however have not been tested extensively.

Client

Browsers

  • Firefox >=31.x (ESR) & 38.x (ESR), limited support for older versions
  • Internet Explorer 10 or higher, limited support for IE8 and IE9
  • Chrome
  • Safari
  • Optional: Silverlight (for PivotView)

Other browsers may also work, however have not been tested extensively.

eCloudManager Supported Provider Versions/Firmwares:

See http://help.fluidops.com/resource/Help:ECMSupportMatrix for the current support matrix

Installation

The Information Workbench supports both Windows and Linux based operating systems.

Windows

Installation with the Windows Installer

Installing the Information Workbench on Windows can be done easily using the Windows installer. During the installation one optionally can select to activate the bundled eDataCenterAnalyzer or eCloudManager App. Note that the installation requires administrator privileges.

You can start/stop the application using the Windows services configuration tool (i.e. Start->Run->services.msc). The Information Workbench service is called fluidOps Platform.

Installation from the zip-distribution

It is recommended to use a 64bit Windows operating system with a 64Bit Java SE Runtime Environment in version 1.8 (taken from the JDK). The reference version shipped with the installer is JRE SE 1.8.0_60 64bit. This is also the version used in steps a) and b) below.

Unpack the distribution into a directory of your choice (e.g. C:\IWB). In the following, we will refer to the absolute pathname of this directory by <IWB_HOME_DIRECTORY>.

Running the Information Workbench as executable

Execute <IWB_HOME_DIRECTORY>/start.cmd

Mac OS X

Please note that, while we have successfully installed and run Information Workbench on OS X, this platform is not officially supported. For OS X, we assume a manual mode of operation by a single user. This means that the software needs to be started and stopped manually from the terminal. Also, in case of multi-user systems, starting and stopping must always be performed by the same user who runs the installation.

To run the Information Workbench on Mac OS X you need

  • A compatible version of the Java runtime (ideally, version 1.8). OS X may ask you whether to install a Java runtime automatically if it detects that one is needed but missing.
  • The distribution of the fluidOps Suite (.zip file).

To get started, proceed as follows:

  • Unpack the distribution .zip file in a folder of your choice.
  • Open the terminal application and cd into the unpacked distribution (e.g., cd Desktop/fluidOps-Suite [ENTER])
  • Run mac-install.sh as a shell script (sh mac-install.sh [ENTER])
  • To start, type ./start.sh [ENTER]
  • After a few seconds, you should be able to access Information Workbench in you browser locally under the address https://localhost:50443

Linux

To run the Information Workbench under Linux a Java SE Runtime Environment version 1.8 (taken from the JDK) must be installed. Note that the Information Workbench does not ship a reference version bundled with the release.

Unpack the distribution into a directory of your choice (e.g. /opt/fluidOps/IWB). In the following, we will refer to the absolute pathname of this directory by <IWB_HOME_DIRECTORY>.

  • Download and install Java SE 64-Bit JDK version 1.8 (http://www.oracle.com/technetwork/java/javase/downloads/index.html).
  • Make sure the java command is added to the command-path of the user root.

Installing and Running as service

  • Create a user under which the iwb shall run, e.g. "fluid" (in the following we will refer to this user as <IWB_USER>).
  • Execute the script linux-install.sh in <IWB_HOME_DIRECTORY> as user root, like follows:
    bash -eu linux-install.sh <IWB_USER>

This installs an init-script as /etc/init.d/fops, sets the ownership of the files to the user <IWB_USER>, puts a file fluidops to the config folder /etc/sysconfig/ (for SUSE-like systems, /etc/default for debian-like systems and /etc/ for others) that holds system wide configuration parameters for the startup process and then starts the application. To make sure this script is executed on reboot create corresponding links in the run-level specific directories. Depending on your distribution, this can be done with: chkconfig -a fops or with insserv fops

The application can be started with the following command:

  • Run /etc/init.d/fops start

Upgrade Notes

The following listing contains noteworthy changes in the 7.1 release

  • It is recommended to use a recent Java 1.8 runtime environment to run an Information Workbench or eCloudManager installation. Particularly, various Java 8 JVM configuration settings are supplied to the JVM (e.g. GC and Metaspace settings). Note that an appropriate Java 8 runtime environment is bundled with the Windows Installer already since a few releases.
  • In the Web-Server the caching of static resources (e.g., images and java scripts) has been increased from 1h to 24. In addition, all java scripts are now gzip compressed.
  • The certificate used by the Jetty HTTP(s) webserver has been updated to use a stronger hash algorithm. Note that this is only applied in fresh installations as the keystore is copied over from the previous installation during upgrade. If no custom certificate is used, the keystore from /etc/keystore can simply be replaced with the new version.
  • The default JVM memory is set to 5GB to adhere the requirements of common installations. If required, this value can be adjusted in the backend-user.conf file. Note that even if the OS does only offer less memory, the JVM has some intelligent behavior to deal with this.

Opening the Information Workbench

Once the Information Workbench is running (startup may take up to a few minutes), it can be accessed at https://localhost:50443 . On the default start page you can find a tutorial with some first steps as well as links to important pages (such as the detailed user documentation).

By default, an administrator account with credentials admin/iwb is created. It is highly recommended to change the password in the Admin area after the first login.

Shutting down the Information Workbench

  • Windows Service: Go to the Services configuration tool and stop the service
  • Linux Daemon: Invoke /etc/init.d/fops stop
  • Commandline: Exit the Information Workbench by clicking in the command line window and pressing Ctrl + c.
  • IMPORTANT: Never exit the Information Workbench by closing the command line window without proper shutdown. This can result in corrupting the data store and loss of data.

Enabling HTTP

By default the system is configured to use the confidential mode (i.e. HTTPS): The application server will automatically redirect each request to the secure confidential HTTPS handler.

Support for HTTP can be enabled by means of the system configuration setting "enableHttp".

Troubleshooting

Where can I find additional information

The Information Workbench is configured to write logging output to <IWB_HOME_DIRECTORY>/logs/. Runtime logs of the Information Workbench can be found in service.log, while output of the wrapper is written to wrapper.log. Please consider these files as a first source for further information. Note that the logging level can be configured in <IWB_HOME_DIRECTORY>/etc/log4j.properties.

If you have any questions, please do not hesitate to contact us.
